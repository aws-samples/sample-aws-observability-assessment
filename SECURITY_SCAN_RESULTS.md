# Security Scan Results and Mitigations

## Scan Tools Used
- **Semgrep**: Static analysis for code security patterns
- **Bandit**: Python-specific security linter
- **Manual review**: Command injection and data flow analysis

## Scan Date
2026-03-26

## Findings and Mitigations

### HIGH/CRITICAL: Command Injection via subprocess (16 instances)

**Finding**: `subprocess.run()` called with dynamically constructed command strings.

**Root Cause**: The assessment tool executes ~70 AWS CLI commands via `subprocess.run()`. Commands are constructed by string formatting (e.g., inserting `--profile` and `--region` flags).

**Risk Assessment**: LOW — All command strings are constructed internally from hardcoded AWS CLI commands. No user-supplied input (HTTP requests, file contents, environment variables from untrusted sources) is interpolated into commands. The `--profile` and `--region` values come from CLI arguments parsed by `argparse`.

**Mitigation Applied**:
1. Default execution uses `shlex.split()` with `shell=False` — prevents shell metacharacter interpretation
2. Only 3 commands (SSM remote checks with pipes/redirects) use `shell=True`, suppressed with `# nosec B602` and `# nosemgrep`
3. The `$(...)` shell subcommand was replaced with a Python variable reference (`self.results.account_id`)

**Residual Risk**: Minimal. An attacker would need to modify the Python source code itself to inject commands, at which point they already have arbitrary code execution.

### MEDIUM: Arbitrary time.sleep() (2 instances)

**Finding**: `time.sleep(3)` calls used to wait for SSM command completion.

**Mitigation Applied**: Replaced with `_poll_ssm_command()` method that polls SSM for command completion status with configurable retry intervals, eliminating arbitrary sleep durations.

### LOW: Missing encoding parameter on open() (3 instances)

**Finding**: `open()` calls without explicit `encoding` parameter.

**Status**: Acknowledged. Files are generated by the tool itself (HTML/CSV) and consumed in the same environment. Risk of encoding corruption is minimal in this context.

### INFO: CSV injection risk (2 instances)

**Finding**: Use of built-in `csv` module instead of `defusedcsv`.

**Status**: Acknowledged. CSV output contains AWS resource metadata (ARNs, counts, status strings) generated by the tool — not user-supplied data. Formula injection risk does not apply.

## Compensating Controls

1. **No external input**: The tool accepts only `--profile`, `--region`, `--role-arn`, `--single-check`, and `--single-question` via argparse. None of these are interpolated into shell commands without validation.
2. **Read-only operations**: The tool only calls AWS describe/list APIs (except SSM SendCommand for agent detection).
3. **No network listeners**: The tool does not open any network ports or accept incoming connections.
4. **No persistent credentials**: Temporary STS credentials are used and discarded after execution.
